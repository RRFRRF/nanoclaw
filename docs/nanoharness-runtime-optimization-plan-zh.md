# NanoHarness 运行时优化计划

## 背景

NanoHarness 是 NanoClaw 的一个分支，它已经将容器端的 Claude Agent SDK 执行路径替换为 Deep Agents。目标产品方向如下：

- 用户通过 TUI 管理智能体 (Agents)
- 用户向这些智能体分发任务
- 智能体在容器内执行长期运行的工作流
- 工作流可能运行数小时
- 在可能的情况下，任务应在瞬时提供商故障中幸存
- 运行时应从特定于 NanoClaw 的工具适配层演进为更通用的、支持 MCP 的架构

本文档记录了下一阶段的优化工作，重点关注三个领域：

1. 针对瞬时提供商错误，在宿主机端对计划任务进行重试
2. 可配置的自动继续 (auto-continue)，包括计划任务
3. Deep Agents 运行时中的通用 MCP 后端集成层

## 当前状态总结

### 已实现功能

- TUI 智能体创建、切换、删除以及直接消息分发
- 基于组 (group) 的容器隔离
- 容器内的 Deep Agents 执行
- 会话 + 检查点 (checkpoint) 持久化
- 来自容器的实时状态流
- 宿主机上的实时日志流文件
- 在运行时中以 `mcp__nanoclaw__*` 形式公开的特定于 NanoClaw 的编排工具

### 待填补的空白

- 计划任务在宿主机层不会针对瞬时的上游故障进行重试
- 自动继续是硬编码的，且对计划任务禁用
- 通用 MCP 服务器尚未接入 Deep Agents；目前仅存在特定于 NanoClaw 的工具适配器

## 优化目标

### 目标 1：计划任务的可靠性

当故障可能是瞬时发生时，为计划任务添加宿主机端重试循环：

- HTTP 429 / 频率限制
- 上游 5xx 错误
- 暂时不可用
- 超时 / 连接重置 / 网关错误

限制条件：

- 避免无限重试循环
- 保留任务日志及其可见性
- 在部分成功的情况下，不要重复发送面向用户的消息

### 目标 2：可配置的自动继续

将硬编码的自动继续行为替换为配置项：

- `NANOCLAW_AUTO_CONTINUE_LIMIT`
- `NANOCLAW_AUTO_CONTINUE_SCHEDULED`

预期行为：

- 默认情况下，普通对话轮次保留当前的自动继续语义
- 计划任务可以自主选择加入同一个继续循环
- 继续限制是明确的且易于理解

### 目标 3：通用 MCP 后端层

为 Deep Agents 引入可重用的 MCP 客户端层：

- 支持 stdio 类型的 MCP 服务器
- 通过 `listTools` 动态发现工具
- 将其作为 Deep Agents 工具与现有的 NanoClaw 工具并列公开
- 保留 `mcp__nanoclaw__*` 以实现向后兼容

限制条件：

- 不要破坏现有的运行时
- 保持工具命名稳定且明确
- 清晰地记录工具加载失败的情况

## 交付计划

### 第 1 阶段：可靠性原语

- 添加计划任务瞬时错误分类
- 添加重试配置和有界退避 (bounded backoff)
- 确保在记录最终任务失败之前进行重试

### 第 2 阶段：继续控制

- 使自动继续可通过环境变量配置
- 配置后允许计划任务参与继续逻辑
- 通过专项测试覆盖相关行为

### 第 3 阶段：通用 MCP 集成

- 添加 MCP 客户端传输引导代码
- 加载配置的 stdio 服务器
- 将 MCP 工具转换为兼容 Deep Agents 的工具包装器 (wrappers)
- 保持现有 NanoClaw 工具并行注册

### 第 4 阶段：验证

- 构建
- 针对性的运行时测试
- 调度器测试
- 针对现有宿主机/容器流程的回归测试

## 实现说明

### 计划任务重试模型

建议的默认策略：

- 重试次数：3
- 基础退避时间：10 秒
- 指数级退避
- 仅在未观察到成功完成标记时进行重试

### 自动继续配置

建议的默认值：

- `NANOCLAW_AUTO_CONTINUE_LIMIT=6`
- `NANOCLAW_AUTO_CONTINUE_SCHEDULED=0`

### MCP 配置模型

初始传输范围：

- 仅限 stdio

建议的环境变量载荷：

- `NANOCLAW_MCP_SERVERS_JSON`

结构示例：

```json
[
  {
    "name": "playwright",
    "command": "node",
    "args": ["./mcp/playwright-server.js"],
    "cwd": "/workspace/group",
    "env": {
      "FOO": "bar"
    }
  }
]
```

工具公开模型：

- 工具名称变为 `mcp__<server_name>__<tool_name>`
- 现有的 `mcp__nanoclaw__*` 工具仍作为原生运行时工具保留

## 风险

### 风险 1：计划任务输出重复

如果在部分结果流式传出后发生提供商错误，重试可能会导致向外发送重复消息。

缓解措施：

- 仅在未发出完成结果时进行宿主机端重试
- 跟踪在尝试期间是否已发送用户可见的结果

### 风险 2：MCP 模式不匹配

MCP 工具公开的是 JSON Schema，而 Deep Agents 工具包装器需要 Zod schema。

缓解措施：

- 为常见的标量/对象/数组类型实现一个有界的 JSON-Schema 到 Zod 的转换器
- 当无法进行精确转换时，回退到通用的 JSON 对象 schema

### 风险 3：额外的运行时复杂性

容器运行时已经承载了兼容性逻辑。

缓解措施：

- 将新的 MCP 代码隔离到助手函数中
- 在不存在 MCP 配置时保留现有行为

## 测试计划

### 计划任务重试

- 针对 429 进行重试
- 针对 502 / 暂时不可用进行重试
- 不会对确定的无效输入进行重试
- 在达到配置的重试预算后停止

### 自动继续

- 遵循基于环境变量的限制
- 默认跳过计划任务
- 启用后包含计划任务

### MCP

- 解析配置的 stdio 服务器
- 列出工具并注册包装器
- 优雅地忽略失败的 MCP 服务器启动
