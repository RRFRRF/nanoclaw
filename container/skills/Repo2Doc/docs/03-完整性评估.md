# 03 — 完整性评估

> 本文件由 `SKILL.md` 第三阶段引用。定义如何评估文档完整性，以及何时停止迭代。

---

## 评估维度

每次完整性检查，从以下 5 个维度评分：

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| **技术原理** | 25% | 核心功能的实现原理是否解释清楚？使用了什么算法/设计模式？为什么这样设计？ |
| **代码映射** | 25% | 主要功能是否都对应到具体代码路径？关键类/函数是否已说明？ |
| **架构可视化** | 20% | 是否有清晰的 Mermaid 架构图？是否有模块依赖关系图和数据流图？ |
| **上手指南** | 15% | 新人是否知道从哪里开始看代码？是否有运行和调试指南？ |
| **实用性** | 15% | 如果要修改某功能，能否快速定位代码？是否有常见修改场景指南？ |

---

## 评估输出格式

每次评估后，在内部生成以下 JSON 结构（不输出给用户，用于决策）：

```json
{
    "is_complete": false,
    "confidence_score": 0.65,
    "dimension_scores": {
        "技术原理": 0.6,
        "代码映射": 0.7,
        "架构可视化": 0.5,
        "上手指南": 0.8,
        "实用性": 0.6
    },
    "missing_parts": [
        "模块A的实现原理未解释",
        "缺少数据流图",
        "核心类 Handler 的设计意图未说明"
    ],
    "next_actions": [
        {
            "action": "读取文件",
            "target": "src/core/engine.py",
            "reason": "需要了解核心引擎的实现原理"
        },
        {
            "action": "搜索代码",
            "target": "grep -r 'class Handler' --include='*.py'",
            "reason": "需要定位 Handler 类了解设计模式"
        }
    ]
}
```

---

## 完成条件

满足以下**全部**条件时，判定文档完成：

1. ✅ 核心模块的实现原理已解释（不只是列出模块名）
2. ✅ 包含至少 1 个有效的 Mermaid 架构图
3. ✅ 主要模块都有代码路径映射（文件 → 功能的对应表）
4. ✅ 有新人上手指南（推荐阅读顺序 + 运行步骤）
5. ✅ 置信度 ≥ 0.85（加权计算）

**置信度计算方式**:
```
置信度 = 技术原理 × 0.25 + 代码映射 × 0.25 + 架构可视化 × 0.20 + 上手指南 × 0.15 + 实用性 × 0.15
```

---

## 继续信号（存在以下任一 → 继续迭代）

```
✅ 核心入口文件尚未读取
✅ 存在重要模块但实现原理未解释
✅ 文档中有 "⚠️ 待确认" 标记可以通过读取代码消除
✅ 缺少 Mermaid 架构图或模块依赖关系图
✅ 置信度 < 0.85
✅ 某个维度得分 < 0.5（短板明显）
```

---

## 停止信号（满足以下任一 → 停止迭代）

```
⏹ 置信度 ≥ 0.85
⏹ 已达到最大迭代次数（6 次）
⏹ 连续 2 次迭代后置信度提升不足 0.05（收益递减）
⏹ 所有核心模块已分析完毕
```

---

## 覆盖维度检查表

迭代结束前，对照此检查表自查：

### 代码结构
- [ ] 项目根目录的结构已解释
- [ ] 每个主要目录的职责已说明
- [ ] 入口文件已定位和分析

### 核心模块
- [ ] 核心模块的入口类/函数已记录
- [ ] 核心模块的实现原理已解释
- [ ] 模块间的依赖关系已说明

### 架构可视化
- [ ] 系统架构图（组件及关系）
- [ ] 模块依赖关系图
- [ ] 核心数据流图
- [ ] 至少 1 个关键流程的时序图

### 实用信息
- [ ] 从哪里开始看代码的建议
- [ ] 如何运行项目
- [ ] 常见修改场景的代码定位指南

---

## 评估后的迭代决策

```
完整性检查结果
    │
    ├─ 置信度 ≥ 0.85 且 5 个条件全满足
    │   → 进入保存阶段
    │
    ├─ 已达最大迭代次数（6 次）
    │   → 强制进入保存阶段（标注未完成的部分）
    │
    └─ 置信度 < 0.85
        │
        ├─ 有明确的 next_actions
        │   → 执行 next_actions 中的文件读取/搜索
        │   → 更新文档
        │   → 再次评估
        │
        └─ 没有明确的下一步
            → 重新审视目录结构
            → 选择尚未读取的重要文件
            → 更新文档
            → 再次评估
```
