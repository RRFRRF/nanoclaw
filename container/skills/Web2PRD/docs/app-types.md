# 应用类型探索策略

> 首页分析后按需读取此文件。根据应用类型选择对应的探索优先顺序。

---

## 分类决策树

```
首页截图 + 快照分析
   │
   ├─ 商品网格 + 价格 + 购物车图标？            → 电商
   ├─ 侧边树形导航 + 代码块/版本选择器？         → 文档/Wiki
   ├─ 侧边图标导航 + 数据表格 + 图表？           → SaaS 控制台
   ├─ 文章卡片 + 日期 + 作者署名？               → 博客/资讯
   ├─ CRUD 表格 + 用户管理 + 权限设置？           → 管理后台
   ├─ Feed 流 + 关注/点赞 + 个人主页？            → 社区/社交
   ├─ Hero 区块 + 定价表 + CTA 按钮？             → 落地页/营销
   ├─ 子域名/路径组织标识 + 工作区切换器？         → 多租户 SaaS
   └─ 无法分类？                                  → 通用策略
```

---

## 1. 电商

**信号**: 商品网格、购物车图标、价格、结账按钮

| 顺序 | 页面 | 要点 |
|------|------|------|
| ① | 首页 | 分类结构、导航 |
| ② | 分类列表页 | 筛选、排序、商品卡片、分页 |
| ③ | 商品详情页 | 标题/价格/规格/加购/评价 |
| ④ | 购物车 | 商品列表/数量/小计/结算 |
| ⑤ | 结账流程 (DFS) | 每步表单字段 |
| ⑥ | 用户账户 | 个人资料/订单列表 |
| ⑦ | 搜索结果 | 结果布局、空结果 |
| ⑧ | 认证页面 | 登录/注册 |

## 8. 多租户 SaaS

**信号**: 子域名 `{org}.app.com`、工作区切换器

| 顺序 | 页面 | 要点 |
|------|------|------|
| ① | 公开落地页 + 注册 | 创建工作区 |

## 通用策略

```
① 首页 → 识别主要内容区域 → 记录所有一级导航
② 每个导航项：访问 → 截图 → 记录新链接
③ 发现表单：记录字段 → DFS 跟随
④ 发现登录入口：参照认证处理
⑤ 发现搜索：执行搜索测试
⑥ 评估覆盖率
```
