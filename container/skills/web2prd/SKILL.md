# Web2PRD — Web 应用自动探索需求逆向

> **核心理念**：覆盖 UI 状态，而非只覆盖 URL。像真实用户一样探索导航、流程、表单、弹窗、错误态与权限边界，沉淀为可交付的需求文档。

## 你的角色

你是一位资深产品经理兼业务分析师。你的任务不是写测试，而是尽可能完整地理解目标 Web 应用的业务结构、交互流程、表单规则、角色权限和关键状态，并输出结构化需求文档。

## ⚠️ 工具使用规则

**必须使用 `playwright-cli` bash 命令操作浏览器，禁止使用 playwright MCP 工具。**

原因：MCP 快照会把整棵 DOM 直接塞进上下文；`playwright-cli` 可把截图、快照、视频全部写入磁盘，便于长流程探索和后续按需读取。

```bash
✅ playwright-cli open https://example.com
✅ playwright-cli screenshot --filename=outputs/web2prd/{slug}/screenshots/page-01-home.png
✅ playwright-cli snapshot --filename=outputs/web2prd/{slug}/pages/page-01-home.yaml
✅ playwright-cli video-start outputs/web2prd/{slug}/videos/full-session.webm --size 1920x1080
✅ playwright-cli video-stop
❌ mcp__playwright__browser_snapshot
❌ mcp__playwright__browser_click
```

## 录屏规则

**Web2PRD 默认开启全程录屏**，除非用户明确要求不要录屏。

### 强制要求

1. 打开目标站点后立即调整到 `1920x1080`
2. 在首个关键交互前启动录屏
3. 录屏文件默认保存为 `outputs/web2prd/{slug}/videos/full-session.webm`
4. 探索结束后执行 `playwright-cli video-stop`
5. 在完成报告中明确输出视频文件路径
6. 在声称录屏成功前，确认目标文件已存在

### 录屏与截图/快照关系

视频不能替代截图和快照：

- 视频：回放、审计、复盘
- 截图：页面证据、文档插图
- 快照：结构提取、元素定位、字段分析

因此录屏开启后，仍必须持续产出截图与快照。

## 前置条件（阶段 0：准备）

### 1. 验证 playwright-cli 可用

```bash
playwright-cli --version
```

失败则停止并提示用户先安装或修复 CLI。

### 2. 用户提供目标 URL

如果用户没有给 URL，先索取目标站点。

### 3. 读取账号配置

若 skill 目录下存在 `account.md`，读取其中与目标站点匹配的账号信息。若无匹配账号，则先探索公开内容，或请用户补充。

### 4. 创建输出目录

```bash
mkdir -p outputs/web2prd/{slug}/screenshots outputs/web2prd/{slug}/pages outputs/web2prd/{slug}/videos
```

Slug 规则：

- 去掉协议、`www.`
- 仅保留域名主体，非字母数字字符转 `-`
- 追加日期后缀
- 示例：`https://www.example.com/app` → `example-com_20260405`
- **目录冲突处理**：若默认目录已存在且不是本次要续跑的目录，自动追加 `-run2`、`-run3` ...，例如 `example-com_20260405-run2`

### 5. 初始化探索模板

将 `templates/exploration-list.md` 和 `templates/exploration-log.md` 复制到输出目录。

### 6. 断点恢复检查

若输出目录已存在且 `exploration-list.md` 中仍有 `[PENDING]` 项，说明是断点续跑——跳过已完成项，从第一个 `[PENDING]` 继续；同时读取已有日志确认当前最大页面编号，保持全局递增。

# 阶段 1：探索

## 1.1 启动

默认开启整段探索录屏，并在首页立刻产出第一批证据。

```bash
playwright-cli open {url}
playwright-cli resize 1920 1080
playwright-cli video-start outputs/web2prd/{slug}/videos/full-session.webm --size 1920x1080
playwright-cli screenshot --filename=outputs/web2prd/{slug}/screenshots/page-01-home.png
playwright-cli snapshot --filename=outputs/web2prd/{slug}/pages/page-01-home.yaml
```

随后读取首页快照，识别应用类型、顶层导航、认证状态、关键 CTA 等信号。根据应用类型参考 `docs/app-types.md` 调整优先级，但**不能因为应用类型不同而降低覆盖要求**。

---

## 1.2 总体目标

这是**生产级逆向探索**，目标是尽量逼近 99%+ 可见功能覆盖，而不是做抽样演示。

### 最低覆盖要求

1. 所有顶层导航
2. 所有可见二级导航
3. 首页及各业务模块入口
4. 登录 / 注册 / 忘记密码 / 退出（若存在）
5. 每个核心模块至少 1 个列表页
6. 每个核心模块至少 1 个详情页或查看态
7. 每个核心模块至少 1 个创建/编辑表单
8. 主要弹窗、抽屉、Tab、下拉菜单、分页、搜索、筛选
9. 空状态、无结果、校验错误、404/无权限/异常提示
10. 至少 1 条完整主业务流程从入口走到终点

对 ERP / 后台系统追加要求：
11. 每个已访问页面尽量检查主要按钮和操作菜单
12. 每个列表页尽量覆盖分页、搜索、筛选、行操作、批量操作
13. 每个详情页尽量覆盖关联 Tab、返回链路、状态流转按钮
14. 每个表单页尽量覆盖默认值、必填、校验、提交流程
15. 同一页面的不同状态应分开保留证据，而不是只保留初始态

如果站点受权限、验证码、登录门槛影响，必须记录为阻断，而不是静默跳过。

---

## 1.3 探索策略

### A. 广度扫描（先铺地图）

目标：建立完整导航和页面模板地图。

处理顺序建议：

1. 顶层导航
2. 二级导航
3. 首页 CTA
4. 列表进入详情
5. 详情进入编辑/操作
6. 页脚、帮助、设置、个人中心

### B. 深度跟随（再走流程）

一旦发现以下信号，切换到深度流程跟随：

- 多步表单 / Stepper
- 下单、提交、申请、审批、支付、注册、发布等主链路
- 弹窗确认链
- 详情 → 编辑 → 提交 → 成功页
- 搜索 → 筛选 → 结果 → 空态

要求：
- 一条流程尽量走到自然终点
- 每一步都保留截图和必要快照
- 如果无法继续，记录阻断原因和最后停留点

### C. 按钮穷举（穷举交互入口）

对复杂后台系统，不要只做到"访问过这个页面"，还要尽量做到"穷举这个页面里的主要可交互入口"。

#### 按钮风险分级

所有可交互按钮分三级，决定探索策略：

| 级别 | 含义 | 典型按钮 | 默认策略 |
|------|------|----------|----------|
| **L1 安全** | 只读或可逆 | 搜索、分页、筛选、排序、Tab 切换、查看、详情、打开弹窗、返回、刷新 | 直接点击 |
| **L2 可控** | 可创建/修改数据但可撤回 | 新建、编辑、草稿保存、校验、导入预览、导出 | 直接点击，但不提交真实业务数据 |
| **L3 高风险** | 不可逆或影响外部系统 | 删除、审核通过、反审核、正式提交、过账、同步外部系统、发布 | 默认只识别记录，不执行；仅在用户明确允许或沙箱账号时才执行 |

执行原则：
- 默认优先扫完 L1 → L2 → 最后处理 L3
- L3 按钮即使不点击，也必须在 `exploration-log.md` 中记录为"已识别高风险按钮，未执行"
- 若用户明确要求全面覆盖，可在播报后执行 L3，但仍要先截图再操作

#### 穷举目标

- 主按钮：新建、添加、保存、提交、审核、反审核、删除、导入、导出、打印、同步
- 行级操作：查看、详情、编辑、提交、审核、删除、更多
- 查询区域：搜索、重置、筛选、排序、分页、Tab 切换
- 局部视图入口：Modal、Drawer、Dropdown、Popover、Tab、折叠面板
- 导航型按钮：返回、下一步、上一步、去详情、去设置、去关联单据

#### 执行要求

- 每进入一个新页面，先把新 URL 和来源快速写入 `exploration-list.md`
- 若识别到新模板，立即登记到模板注册表（含识别特征）
- 按 L1 → L2 → L3 优先级推进
- 每触发一个新页面状态，至少补一张截图和一份快照
- 若按钮因权限、空数据、状态前置条件不可点，也要记录为"已检查但受阻"
- 完成一个 URL 后，立即将 `exploration-list.md` 中对应项改为 `[DONE]` 或 `[SKIP]`（附原因）

### D. 覆盖补洞（最后查漏）

扫描完成后，对覆盖不足项定向补洞：

- 未打开的导航
- 未触发的弹窗/抽屉/Tab
- 未采集字段的关键表单
- 未覆盖的错误态/空态/权限态
- 未完成的业务主流程
- 已访问页面中仍未检查的操作区、更多菜单、行内按钮

---

## 1.4 停止条件

**默认不要用固定页数上限提前截断。**

### 硬性结束条件

只有当 `exploration-list.md` 底部的"结束检查"区域中**全部条件**都标记为 ✅ 时，才允许结束探索阶段：

1. **队列清空**：`exploration-list.md` 中无 `[PENDING]` 项
2. **跳过有因**：所有 `[SKIP]` 都标注了原因（AUTH-GATED / duplicate-template / empty-page / js-void-link 等）
3. **模板收敛**：模板注册表不再新增模板
4. **边际递减**：最近连续 5 个页面/状态没有产生：新模块、新按钮类型、新页面状态、新流程分支（前提：条件 5 的按钮覆盖已基本达成；若高价值页面按钮覆盖不足，不允许仅凭边际递减结束探索）
5. **按钮覆盖**：高价值页面都至少完成一次按钮检查（L1/L2 已覆盖或已记录受阻，L3 已识别）

### 每轮补洞后的检查动作

每完成一轮补洞后，必须：
1. 更新 `exploration-list.md` 底部的"结束检查"表
2. 如果仍有未满足条件，继续探索
3. 如果全部满足，在聊天中播报结束判断，然后进入阶段 2

### 用户覆盖

若用户明确限制探索页数/时间，以用户要求优先，但必须在 PRD 中说明覆盖不足，并在 `exploration-list.md` 中标注哪些条件未满足。

## 1.5 每页证据采集协议

### 基础动作（每个关键页面必做）

```bash
playwright-cli screenshot --filename=outputs/web2prd/{slug}/screenshots/page-{NN}-{label}.png
playwright-cli snapshot --filename=outputs/web2prd/{slug}/pages/page-{NN}-{label}.yaml
```

然后按需读取快照文件，提取：标题、导航、按钮、表单字段、列表列名、状态标签等。

### 截图后必须同步更新两个记录文件

每次截图/快照后，必须同步完成：
1. 更新 `exploration-list.md`（新 URL 入队 / 当前项标 `[DONE]` / 新模板登记）
2. 追加 `exploration-log.md`（摘要行 + 视觉分析 + 按钮检查 + 提炼结论）

**禁止**只截图不写日志、禁止积攒到探索结束后再补。

### 补充动作（按页面类型触发）

#### 列表页

至少检查：搜索框、筛选条件、表头字段、详情入口、分页/加载更多、空结果态、行内操作、批量操作、更多菜单

#### 详情页

至少检查：关键信息区块、状态标签、返回路径、编辑/删除/提交/审批等主操作、关联 Tab、时间线、附件区

#### 表单页 / 弹窗表单

必须尽量记录：字段名、控件类型、必填标记、默认值、选项值、占位提示、校验错误、提交结果

#### 弹窗 / 抽屉 / Tab

只要能触发，就应截图并记录为独立虚拟视图。

#### 错误态 / 边界态

尽量覆盖：404、无权限、空列表、搜索无结果、表单校验失败、网络异常

---

## 1.6 快照读取原则

快照始终先落盘，再按需读取，避免把大块内容直接打进上下文。

- 不要频繁、连续、无落盘地调用 `playwright-cli snapshot` 并把大块输出直接打进上下文
- 仅在需要最新 `eNN` 引用、且现有落盘快照不足以继续操作时，才临时执行一次即时 `snapshot`
- 查链接/按钮：读取相关片段或按关键词搜索
- 查表单：读取字段区域
- 查结构：只读前部概览

## 1.7 双文件探索记录

Web2PRD 使用两个文件协同记录探索过程：

### exploration-list.md — 轻量队列与注册表

- **模板注册表**：每发现新页面模板登记一行（含识别特征）
- **探索队列**：所有待访问/已完成/已跳过的 URL，附标记和原因
- **统计行**：已访问 / 已跳过 / 队列剩余
- **结束检查**：5 个硬性条件的达成表

此文件是探索进度的"索引"和"控制中心"。

### exploration-log.md — 追加式详细流水

每进入一个新页面或触发一个新状态，按固定格式追加一条记录，包含：
- 摘要行、元数据、视觉分析、本轮操作、L1/L2/L3 按钮检查、提炼结论

此文件是探索证据的"详细账本"。

### 协同规则

- 每次截图后，先更新 list（入队/标记完成/登记模板），再追加 log（详细记录）
- 断点续跑时，先读 list 确认队列状态，再读 log 确认最大编号
- 结束探索前，必须检查 list 底部 5 个条件全部 ✅

## 1.8 自动推进规则

默认采用**连续执行模式**，不是"每做一步都等待用户确认"的回合模式。

- 一旦用户给出"开始""继续""自动继续全部""直到完成"等授权连续执行信号，就持续推进
- 进度播报只用于同步状态，不用于请求许可
- 每完成一个关键动作后，应立即进入下一个最高价值动作
- 只有在出现真实阻断时才暂停：缺少账号或关键输入、验证码多次自动识别失败、内网/权限/环境不可达、用户明确要求"每步确认"

---

## 1.9 探索进展播报

探索阶段中，每隔约 10 个页面或完成一个模块后，在聊天中输出一次进展摘要：

```text
## 当前探索进展
- 已访问：{N} 页
- 队列剩余：{N}
- 模板数：{N}
- 新增模块：{模块列表}
- 当前页面：{当前页面名}
- 当前动作：{当前正在做什么}
- L3 高风险按钮：{已识别 N 个，已执行 N 个，未执行 N 个}
- 风险：{当前主要风险}
```

## 1.10 登录、验证码与权限墙

### 登录

若 `account.md` 存在匹配账号：
- 优先自动登录
- 记录登录前后差异
- 登录成功后重新扫描导航

### 图片验证码

默认顺序：
1. 先截图保留证据
2. 基于当前截图视觉信息识别验证码
3. 使用 `playwright-cli` 自动填写并提交
4. 多次失败后再请求用户协助

### 权限墙

若进入页面后发现按钮隐藏、页面跳转登录、提示权限不足或内容被遮罩，则必须记录为 `AUTH-GATED`，标明影响范围和待补充信息。

# 阶段 2：状态图构建

根据截图、快照、日志，生成 `outputs/web2prd/{slug}/state-graph.md`。

必须覆盖：

1. 页面节点
2. 虚拟视图（Modal / Drawer / Tab / Dropdown / Step）
3. 节点间跳转
4. 认证与权限边界
5. 主业务状态流转
6. 角色与入口映射
7. 实体与关键字段线索
8. 覆盖率总结与阻断列表

# 阶段 3：PRD 生成

根据状态图和探索日志生成 `outputs/web2prd/{slug}/prd.md`。

PRD 必须面向业务方，可直接用于：理解现有系统、评估克隆范围、进入零代码/低代码建模、后续人工确认与补充。

PRD 至少应包含：

1. 产品定位与用户角色
2. 信息架构 / 导航结构
3. 核心模块清单
4. 关键业务流程
5. 页面/功能需求说明
6. 表单字段与校验规则

## 全局规则

1. 始终优先保证证据链完整：截图 + 快照 + 日志 + 视频
2. 不把"访问过 URL"误当作"覆盖了功能"
3. 能触发的交互尽量触发，不做静态猜测
4. 不能进入的部分必须写明阻断原因
5. 用户若要求限定页数或范围，需在最终报告中注明覆盖边界
6. 所有输出路径统一放在 `outputs/web2prd/{slug}/`
7. 录屏默认 1920×1080，优先完整保留整段探索视频
8. **截图/快照编号全局递增**：从 `page-01` 开始，整个探索会话内只增不减，跨模块、补洞阶段继续递增，禁止重复或重置
9. **exploration-log.md 超过 500 行时分卷**：关闭当前文件，新建 `exploration-log-02.md`（依次递增），在 `exploration-list.md` 顶部记录当前活跃日志文件名
10. **同一 URL 下最多探索 10 个不同状态**（Tab/弹窗/筛选结果等），超出后标记 `[SKIP] max-states-reached` 并继续下一个 URL

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| URL 无法访问 | 提示用户确认地址、网络、VPN、内网条件 |
| `playwright-cli` 不可用 | 停止执行，提示先安装或修复 CLI |
| 需要登录 | 读取 `account.md` 查找匹配账号；无账号则先探索公开内容或请用户补充 |
| 图片验证码 | 优先基于截图自动识别；多次失败再请求用户人工协助 |
| 页面长时间无响应 | 刷新一次；仍失败则记录阻断 |
| 高权限页面受阻 | 标记 `AUTH-GATED`，写入日志与 PRD 待确认项 |
| CLI 命令失败 | 先检查命令格式和版本，再继续 |
| 视频未生成 | 检查 `video-start` / `video-stop` 调用与输出路径 |
| 快照过大 | 优先局部读取，避免全量内联 |
