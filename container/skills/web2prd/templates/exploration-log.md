# 探索详细流水账

> 按探索顺序持续追加。每进入一个新页面或触发一个新状态后，立即把信息追加到文末。

---

## 探索会话信息

- **开始时间**: {YYYY-MM-DD HH:mm:ss}
- **结束时间**: {YYYY-MM-DD HH:mm:ss}
- **目标 URL**: {url}
- **应用类型判断**: {SaaS / 管理后台 / 电商 / 营销站 / 社区 / 工具型站点 / 未知}
- **探索范围**: {公开内容 / 登录后内容 / 指定模块 / 全站可见范围}
- **输出目录**: `outputs/web2prd/{slug}/`

---

## 追加规则

每个页面或状态都按以下格式直接追加到文末：

```markdown
---

### PAGE-{NN}: {页面名称或状态名称}

> [摘要] {页面类型} | {所属模块} | {已触发的关键操作} | {新发现要点} | {待测按钮}

- 时间: {HH:mm:ss}
- URL: {/path}
- 来源: {从哪个页面/按钮进入}
- 页面类型: {List / Detail / Form / Modal / Tab / Error / Other}
- 截图: `screenshots/page-{NN}-{label}.png`
- 快照: `pages/page-{NN}-{label}.yaml`

**视觉分析**: {页面主区域} | {当前状态：初始/编辑/弹窗/空/报错} | {核心动作}

**本轮操作**:
- {具体动作} → {结果}
- {具体动作} → {结果}

**按钮检查**:
- L1 安全 已覆盖: {搜索/分页/筛选/Tab/查看/详情/打开弹窗}
- L2 可控 已覆盖: {新建/编辑/草稿保存/校验}
- L3 高风险 已识别: {删除/审核/反审核/正式提交/同步外部系统} → {已执行/未执行，原因}
- 受阻: {按钮名 → 原因}
- 待后续: {按钮名}

**提炼**: {页面业务作用} | {对状态图：节点/边/虚拟视图} | {对 PRD：模块/流程/字段/规则}
```
